Gary Marcus（♥12,675 / 70万阅读）「If enough other companies report the same, the bubble pops. 🫧」——当巨额 AI 用量与低产出碰撞，裂痕开始显现
2026-05-24，NYU 教授、著名 AI 批评家 @GaryMarcus 刷屏爆帖。核心事实：① 麦当劳/Uber COO Andrew Macdonald（♥2,228 原帖）表示：Uber 几乎在几个月内就吹爆了全年的 AI 预算，但「没有看到同等比例的生产力提升」，正感到受挫和考虑收窄预算；② Microsoft 砍掉 Claude Code 授权：Tom Warren 在 The Verge 报道，微软近日停止了部分内部开发组的 Claude Code 订阅授权，「因为用量开销过大，超额账单超出了评估收益」；③ 星巴克（Starbucks）关闭 AI 库存系统：Reuters 报道，在部署 9 个月后，星巴克宣布终止 AI 自动库存统计程序，因为该系统在真实复杂的门店环境下经常「算错数量、标记错产品（幻觉在物理环境的代价）」。

Marcus 分析：三大没有盈利的 AI 公司正在密谋合计高达 4 万亿美元市值的超级 IPO，并迫使整个市场指数基金接盘，但这套逻辑建立在「用户对 token 需求是无限的」这一幻想之上。一旦第一批大公司客户开始因为 ROI 差而收缩预算，这个泡泡就会开始爆裂。
这是 2026 年上半年 AI 浪潮最真实的裂痕信号大汇聚。作为 VC：不要相信「AI 自动改变一切企业」的伪装叙事。要相信：① 可验证的刚需场景（如 Microsoft MDASH 找 0-day、100% 成功）；② 不依赖通用 API 账单的本地推论（Localmaxxing）；③ 零幻觉容忍的垂直系统。这是下阶段投资的洗牌方向。

steipete · cmux + 三分工具栈（codex mac / cursor / codex cli）
@steipete ♥3,197 · 2026-05-22 · 专业 AI 开发者工作流参考
工具栈
steipete 分享了他的实际工具分工：Codex Mac App（看着运行中的 Agent），Cursor（交互式编码），Codex CLI（并行批量任务）。配合 cmux 管理多路终端，实现「多 Agent 并行」的完整工作流。这是目前公开描述最清晰的专业 AI Coding 工具组合方案。

三个工具的分工逻辑：

① Codex Mac App：Dashboard 角色——你不用写代码，但你要知道 Agent 在做什么、有没有卡住、需不需要 approve。Mac App 提供了最清晰的「当前 Agent 状态」视图。

② Cursor：「需要人实时参与」的编码任务——代码设计讨论、复杂 debug、需要你立刻给反馈的工作。Cursor 的 codebase-aware 自动补全在这个场景下最有效。

③ Codex CLI：「批量、并行、后台跑」的 Agent 任务——写测试、重构函数、扫描 bug、生成文档。你可以开启 10 个 CLI 实例同时跑不同任务，cmux 帮你管理这些窗口。

cmux 是什么：一个终端多路复用工具（类似 tmux），让你可以在一个终端窗口里管理多个会话，每个 Codex CLI 实例跑在独立的 session 里，随时可以切换查看进度。

这个工作流的隐含条件：steipete 在 OpenAI 工作，有超额的 API 配额。对普通开发者，Claude Code $200/月 的 rate limit 是瓶颈——但这个工具分工的逻辑是通用的。

🔬 技术与学术前沿
Gemini Omni 技术深度
全模态统一理解 vs 多模态管道：为什么这个区别改变了视频 AI 的可能性边界
@emollick 的分析引发了对「什么是真正的多模态」的技术讨论。关键区别：管道式（大多数现有多模态 AI）= 各个模态各自有独立的编码器，然后在 late fusion 阶段合并；统一式（Gemini Omni）= 所有模态在同一个 transformer 的注意力空间里相互影响。后者能处理的任务远超前者：比如「这段视频里，说话人的语气和内容是否矛盾？」这种需要同时理解声音、文字、视觉表情的任务。
真正的全模态统一理解是「世界模型」的核心技术要求。Agora-1（Odyssey ML）和 Gemini Omni 都在这个方向——谁先做到可靠的「统一多模态推理」，谁就有了构建真实 AI Operating Layer 的底层能力。

媒体报道
OpenAI claims it solved an 80-year-old math problem — for real this time — TechCrunch
TechCrunch · 2026-05-20
TechCrunch 对 OpenAI Erdős 突破的最完整报道，标题里的「for real this time」是对 7 个月前同类声明被打脸的直接回应。这次数学家们主动公开背书验证，说明结果是真实的。
▸平面单位距离问题（1946）：n 个点最多有多少单位距离对？80 年以为方格排列最优 (~n^{4/3})，OpenAI 模型发现整族新构型推翻了这个信念
▸关键：通用推理模型（非专门数学模型），数学家 Noga Alon、Melanie Wood、Thomas Bloom 公开背书验证
▸重要说明：整个更大的 Erdős 问题仍未完全解决，这是子问题突破——但依然是 AI 自主数学研究的里程碑
「For real this time」— 背景故事

7 个月前，OpenAI 前 VP Kevil Weil 发推称「GPT-5 解决了 10 个 Erdős 问题」，Thomas Bloom（维护 Erdős Problems 网站）当时称之为「a dramatic misrepresentation」并公开纠错。这件事让业界对 OpenAI 数学声明持高度怀疑态度。

这次对不对：OpenAI 发布了伴随公告的数学家证词，Noga Alon（以色列理工，组合数学权威）、Melanie Wood（哈佛，数论权威）、以及 Thomas Bloom（此前纠错的那位）都主动验证了结果的真实性。这个「三重数学家背书」是 OpenAI 在上次事故后建立可信度的具体措施。

数学内容（简化版）

问题：在平面上放 n 个点，最多有多少对点之间的距离恰好等于 1？直觉答案：把点排成方格，相邻点距离为 1，这样的距离对数约为 n^{4/3}（即 n 的 4/3 次方）。

OpenAI 的发现：存在一整族新的构型，这些构型不是方格，但能实现比 n^{4/3} 更多的单位距离对。这推翻了「方格是直觉最优解」的信念，虽然没有完全解决整个问题（最优解的精确量级仍然未知），但开辟了新的研究方向。

@sama 的帝国叙事
「a general-purpose model solved a major open problem in mathematics. we'll be saying this a lot over the coming years.」

这句话是本期 OpenAI 最重要的产品叙事之一——sama 在告诉所有人：AI 解决数学问题只是开始，未来两三年这种「AI 突破经典问题」的新闻会越来越频繁。这是在为 OpenAI 的 IPO 叙事服务：「我们不只是聊天机器人，我们在推进人类的科学边界。」
